\# PS7 ScriptDesk Privacy Policy



Effective date: May 11, 2026



PS7 ScriptDesk is an ISE-style scripting tool for PowerShell 7.x.



\## Information the app accesses



PS7 ScriptDesk may access files, folders, scripts, settings, PowerShell command metadata, and local paths that the user opens, creates, edits, runs, or configures in the app.



The app may store local settings, caches, temporary script snapshots, diagnostic logs, and editor metadata on the user's computer.



\## Information collected by the developer



PS7 ScriptDesk does not automatically collect, transmit, sell, rent, or share personal information with the developer.



Scripts, file contents, local paths, logs, settings, and command history remain on the user's computer unless the user chooses to share them manually for support or troubleshooting.



\## Diagnostic logs



PS7 ScriptDesk may create local diagnostic logs to help troubleshoot app behavior. These logs may include technical details such as app version, error messages, command metadata status, file paths, timing information, and workflow events.



Diagnostic logs are stored locally on the user's computer. They are not sent to the developer automatically.



\## Internet access



PS7 ScriptDesk does not require an online account to edit or run local PowerShell scripts.



If future features add online services, this privacy policy will be updated to describe what information is accessed, collected, transmitted, stored, or shared.



\## Third-party services



PS7 ScriptDesk may rely on locally installed Microsoft components such as PowerShell 7.x, .NET, and Microsoft Edge WebView2 where applicable.



Use of Microsoft software and services may be governed by Microsoft's own terms and privacy statements.



\## User control



Users control which scripts and files they open, edit, save, and run.



Users may delete local logs, caches, and temporary files from the app's local data folders or through app settings if available.



\## Contact



For support or privacy questions, contact the developer through the support link provided on the Microsoft Store listing.
